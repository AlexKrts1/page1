Терминал сбора данных


    Использовать терминал сбора данных (ТСД) - активация возможности. Настройки справа окна демонстрируют типовые характеристики связи, их можно изменять.

    Программировать предполагаемое количество - добавлять в терминал поле с количеством товара на складе. Если Вы не хотите, чтобы персонал, проводящий переучет терминалом видел остатки - отключите эту возможность.

    Перепрошивка задачи - программирование системных настроек и формата хранимых данных терминала как устройства, а ключевое поле задает под какие группы товара мы оптимизируем переучет: если штрихкод - штучная продукция, если код - весовая. Возможно также применение смешанного режима, когда на товары не имеющие штрихкода выгружается короткий код, а со штрихкодам - собственно сам штрихкод. Подробнее см. в разделе "Переучет". Смешанный режим с добавлением новых товаров используется для проведения прихода товара при помощи ТСД, хотя может использоваться и в переучете. Неоьходимые файлы прошивок находятся в одном каталоге с программой SuperMarket в файлах с расширением .AGX

    Не проставлять "факт" на товарах с совпадающим штрихкодом при загрузке из терминала (кроме первого) - установите, если в справочник товаров по ошибке внесены дубликаты штрихкодов.

    Разрешить приход товара с помощью терминал сбора данных (для новой накладной) - если установлено, то активирует спец.меню и отдельное окно в приходной накладной с помощью которого можно оформлять приходы из ТСД. Требует прошивки терминала в "Смешанном режиме с добавлением новых товаров". Более подробно см. в разделе "Приходная наклданая". Также стает активным загрузка товаров конкретного поставщика в справочнике контрагентов перед проведением такого прихода.
