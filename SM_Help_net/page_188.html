Поиск ошибок

    "Не ошибается тот, кто не работает" - народная мудрость. В процессе работы персонала с программным обеспечением возможны ошибки. Они могут быть разными, например, ошибки ввода информации или ошибки возникающие при аппаратных и программных сбоях и т.д. В программе уже предусмотрены встроенные средства предотвращения многих ошибок, например, в настройках есть опции контролирующие дублирование данных при заполнении карточек товаров, опции восстанавливающие документы, которые были "потеряны" при неожиданном отключении электропитания и т.д. Однако, эти опции могут быть по незнанию или умышленно отключены и ошибки внесены в базу данных.
    Частью средств для борьбы с ошибками является отдельная группа отчетов "Поиск ошибок". Остановимся подробнее на каждом отчете.
    1. Признаки товаров. Служит для обора в виде отчета-списка товаров с установленными заданными признаками, причем можно установить несколько сразу, чтобы провести поиск по совпадениям таких сочетаний. Например, нужно найти товары которые отмечены как весовые, но одновременно имеющие штрихкод единицы товара, что неправильно. Отобрав такие данные, можно исправить справочник, пройдясь по таким товарам. Кроме того, данный отчет может играть просто информационную роль.
    2. Повторяющиеся записи. Поиск ошибок ввода штрихкода товаров, ошибок переброски товаров между весами с печатью, когда в одни весы предпринимается попытка программировать один номер клавиши PLU по двум товарам и, естественно, в весах будет только один из этих товаров, а второй опереаторы на весах не смогут использовать.Для использования отчета выберите нужный вариант отбора, а для поиска одинаковых номеров в весах еще выберите весы, по которым осуществлять поиск.

    3. Перепроводка себестоимости в чеках. На данный момент опция отключена. Функции учета расхождений в стоимости партий и чеков выполняются математикой подсчета в итоговых отчетах по суммовым остаткам.
    4. Перепроводка минусовых партий. На данный момент опция все еще включена, однако для данной функции настоятельно рекомендуется использовать документ переучет, сформированный по партиям (см.разделы справки "Настройки/Наценка-Переучет" и "Документы/Переучет").
    Может формироваться как по товарной группе, так и по всем группам сразу. Склад указывать обязательно. Флажек "Корректировать также журнал чеков" должен быть установлен - при этом будет создаваться документальное подтверждение корректировки-упорядочения остатков по партиям в виде особого чека. Сумма количеств в таком чеке равна нулю, а вот стоимость такого чека отлична от нуля, т.к. партии могут иметь разную цену. В журнале чеков он выделяется красным цветом. Его наличие дает возможность распровести его, откатив назад изменения сделанные перепроводкой. Сумма по таким чекам выводится в "Товарно-денежном отчете 2" отдельной строкой.
    5. Перепроводка итоговых сумм чеков. Укажите диапазон дат начала/конца периода. Укажите минимальный разброс разницы между розничной суммой заголовка чека и вновь подсчитываемой суммой поля розн.сумма со скидкой табличной части чека. По умолчанию, значение равно 0,10грн.Данная опция, в некоторых случаях, позволяет получить "Товарно-денежный отчет 2" с большей точностью.
