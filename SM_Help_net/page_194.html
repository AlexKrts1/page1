Вкладка Дизайнер

    Вторая вкладка дизайнера используется для работы с текстом запросов, их построением, запуском. Вкладка состоит из четырех частей, рассмотрим их подробнее.
    1. Панель инструментов. Обеспечивает сервис для работы с текстом запросов - открыть, закрыть, очистить запрос, позволяет вставить команду SQL из списка, знаки равенства, а затем выполнить запрос и если необходимо, подтвердить транзакцию, делая изменения в базе данных необратимыми (в случае UPDATE и DELETE запросов). Кнопка "Обновить поля" перечитывает структуру базы данных в окно справа от текста запроса.
    2. Список таблиц и полей базы данных. Находится справа от окна текста запроса. При открытии дизайнера автоматически формирует дерево, где заголовками являются таблицы, а подчиненными записями - поля каждой таблицы базы данных. Двойным кликом на заголовке можно вставить имя таблицы, обрамленное пробелами, в текст запроса, а двойным кликом на имени поля - вставить "имя таблицы.имя поля", например, TBL_DIVISION.NAME.
    3. Опции внизу окна дизайнера. Рассмотрим их подробнее:
    При наличии записей переходить на вкладку "Результат" - если запрос построен на базе команды SELECT и количество отобранных записей больше нуля, то при установленной опции после выполнения автоматически станет активной и видимой вкладка "Результат".
    Запрашивать подтверждение транзакции в невозвращающих данных запросах - если запрос построен на базе команды UPDATE или DELETE, то после успешного запуска/выполнения появится вопрос о подтверждении транзакции. Нажатие ОК сделает изменения необратимыми, нажатие Отмена - вызовет откат транзакции.
    Запрашивать сохранение открытого запроса при открытии/создании нового запроса - помогает случайно не стереть предыдущий запрос при открытии старого или при нажатии на кнопку "Новый запрос".   
    4. Окно редактирования запроса. Автоматическая подсветка команд SQL. Контекстное меню, которое можно вызвать нажатием правой кнопки мыши в нужном месте запроса, позволяет вставить текст записи одного из справочников.
