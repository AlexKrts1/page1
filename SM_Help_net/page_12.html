Вкладка Общие


    Путь к базе данных - укажите имя компьютера на котором расположена база данных и через двоеточие укажите адрес базы данных.

    Рисунок главной формы программы - укажите путь и имя изображения (по умолчанию диалог после нажатия кнопки с троеточием справа от поля ввода откроет папку в которую установлена программа). Приложение поставляется с некоторым количеством картинок (в папке в которую установлена программа есть папка LOGO и в ней расположены картинки), которые можно разместить на главной форме.

    Автомасштаб картинки - при включенной опции размер картинки автоматически подстраивается под размер формы, т.е. когда пользователь растягивает или уменьшает размер главного окна, фоновый рисунок претерпевает те же изменения.

    При изменении у товара признака товарной группы или у товарной группы признака отдела - изменять и журнал чеков - автоматическое обновление журнала чеков (необходимо для работы некоторых отчетов)

    Запускать только одну копию программы - данная возможность может быть полезной на рабочем месте кассира и/или операциониста, где есть периферийные устройства такие как принтеры чеков, сканеры и пр., подключаемые через последовательный COM-порт. Установив этот флажок, можно избежать сообщений о том, что порт уже открыт и столкнуться с неработающим устройством только из-за того, что кассир по ошибке два раза запустил программу и первая копия уже захватила все устройства, а кассир видит лишь вторую открытую копию. Однако, в случае с бухгалтерами и менеджерами, напротив, может быть очень удобным открывать по нескольку копий одновременно при длительном сравнительном анализе информации.

    Разрешить работу с тарой - при включенной опции стает доступным добавление в справочник товаров помеченных как тара и их дальнейший возврат поставщикам при работе с возвратной тарой.

    Контроль товара со штрихкодом - не весовой - при включенной опции при сохранении товара происходит дополнительная проверка не отмечен ли он одновременно весовым и единичным со штрихкодом. Рекомендуется устанавливать для сокращения ошибок операторов.

    Использовать временный чек, счет, перемещение, акт переработки - в папке \Temp в корне каталога программы SuperMarket.exe будут создаваться временные файлы для защиты информации от сбоев аппаратной и программной среды. При аварийном завершении работы (например, пропадание электропитания) - наиболее критичные документы будут восстановлены и пользователь возвратится к работе без потери информации.

    При сохранении/изменении товара проверять штрихкод на дубликаты - выдает предупреждение при попытке сохранить товар с уже существующим штрихкодом.

    При сохранении/изменении товара проверять наименование на дубликаты - выдает предупреждение при попытке сохранить товар с уже существующим наименованием.

    Вести LOG-файл - в каталоге с SuperMarket.exe будет создан и будет заполняться файл sm.log, в который будут записываться все важные события в ПО, например, попытки удаления или редактирования каких-то записей или операций персоналом, ошибки и т.д. Впоследствии можно скопировать/переименовать файл в Excel (расширение .xls) и там накладывать фильтры на типы документов и операций для поиска событий и расследование ошибок.

    Название предприятия на ценнике - в данное поле можно ввести до 30 символов - название предприятия, которое печатается на ценниках, обычно прикрепляемых на полках в торговом зале магазина.

    Юридическое название предприятия - в данное поле можно ввести до 30 символов - название предприятия, которое выводится в печатных формах некоторых документов (возвратов поставщику, счете и т.д.)

    Показывать некоторые SQL-запросы в консоли - вывод содержимого некоторых SQL-запросов к базе данных в специальную консоль. Средство тестирования, поиска ошибок и написания новых запросов-отчетов. Это будет более подробно рассмотрено в следующем разделе справки.

    Показывать форму ожидания при длительных операциях - при выполнении длительных отборов данных, печати чеков и пр. на экран выводится форма с анимацией для того, чтобы пользователь видел, что выполнение программы продолжается.

    При отображении справочника товаров с остатками выделять цветом товары с наценкой товара < наценки товара/группы - эта опция имеет значение только для указанного режима отображения справочника товаров. Красным штрифтом отмечаются товары с наценкой последней партии меньше наценки по умолчанию в карточке данного товара, а пурпурным цветом показываются товары с наценкой последней партии меньше наценки по умолчанию товарной группы данного товара. 
    При отображении справочника товаров с остатками отбирать закупочную цену минимальную по всем складам - эта опция имеет значение только для указанного режима отображения справочника товаров.
    При открытии справочника товаров не обновлять товары сразу, а ждать поиска или обновления вручную - опция очень ускоряет работу программы при больших размерах справочника товаров и базу данных. При добавлении товаров в чек или в накладные не будет паузы при перечитывании всего справочника с сервера.
      А также при этом не сбрасывать фильтр/курсор при добавлении по коду в приходную накладную - работает при установленной предыдущей опции и еще больше ускоряет работу при приходе однотипного товара.
